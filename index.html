Privacy Policy — Hessati

Last updated: July 23, 2026

Hessati ("the app") is an offline attendance and payment tracker for private tutors and small-class teachers. This policy explains what data the app handles and, just as importantly, what it does not.

Summary

  • Hessati works fully offline. All group, student, attendance, and payment data you enter stays on your device, in the app's local database.
  • We do not operate an account system, and there is no cloud sync of your teaching data.
  • The app makes exactly one type of network call — to validate your access code or free trial — described below. No student or group data is ever included in it.
  • Hessati contains no advertising and no third-party analytics or tracking SDKs.

Data you enter into the app

The following information is stored locally on your device only, in the app's own database:

DataPurposeWhere it's stored
Group namesOrganize students into classes/cohortsOn-device only
Student names and optional phone numbersIdentify students; tap-to-dial a parent/studentOn-device only
Attendance records (session dates, present/absent)Calculate each student's balanceOn-device only
Payment records (sessions covered, date)Calculate each student's balanceOn-device only
App settings (threshold, language, theme)Personalize the appOn-device only

None of this data is transmitted anywhere unless you deliberately export it (see "Backup and export" below). We, the developer, never receive it and have no way to access it remotely.

Backup and export (your action, your choice)

Hessati lets you export a full backup (JSON) or a spreadsheet (CSV/Excel) of your data using Android's standard file picker. You choose the destination — local storage, an SD card, a cloud drive, email, etc. This is entirely under your control; the app itself does not upload this file anywhere. Importing a backup replaces the app's local database with the file's contents, again entirely on-device.

Access code / free trial validation (the one network call)

Hessati requires a developer-issued access code (or a one-time 30-day free trial) to unlock the app, since it's distributed directly by the developer rather than through a Play Store subscription. To check whether a code or trial is valid, the app contacts our validation server with only:

  • The access code you entered (or a trial-start request), and
  • A device identifier, used solely to enforce one code/device (or a limited number of devices for multi-device codes) and to prevent a single device from claiming more than one free trial.

No group, student, attendance, or payment data is included in this call, ever. During an active free trial or a previously validated code, everyday use of the app remains fully offline except for this periodic re-check.

Permissions the app requests

  • Notifications — to alert you locally when a student crosses your configured balance threshold. This is a local, on-device notification; nothing is sent to a server to trigger it.
  • Storage / file access (via the system file picker) — only when you choose to export or import a backup or spreadsheet.
  • Phone / dialer — only when you tap a student's phone number to place a call; the app does not read your call log or contacts.
  • Network — used exclusively for the access-code/trial validation call described above.

Data sharing

We do not sell, rent, or share your teaching data (groups, students, attendance, payments) with any third party, because we never receive it in the first place. The only data that ever reaches our server is the access code and device identifier described above, used solely to operate the licensing system.

Data retention and deletion

Your teaching data lives in the app's local database until you delete it — by archiving or deleting groups/students within the app, or by uninstalling the app, which erases its local database entirely (unless you have exported a backup, which is your own separate copy). Access-code/device records on our validation server are retained only for as long as needed to administer your subscription or trial.

Children's privacy

Hessati is a tool for teachers/tutors, not a child-facing app. A teacher may enter a minor student's name and phone number for their own record-keeping purposes; this data is handled the same as any other data described above (stored locally, never transmitted by us).

Changes to this policy

If this policy changes, the "Last updated" date at the top of this page will be revised. Continued use of the app after an update constitutes acceptance of the revised policy.

Politique de confidentialité — Hessati

Dernière mise à jour : 23 juillet 2026

Hessati (« l'application ») est une application hors-ligne de suivi de présence et de paiements pour les professeurs particuliers et les enseignants de petits groupes. Cette politique explique quelles données l'application traite et, tout aussi important, celles qu'elle ne traite pas.

Résumé

  • Hessati fonctionne entièrement hors ligne. Toutes les données de groupes, d'élèves, de présence et de paiements que vous saisissez restent sur votre appareil, dans la base de données locale de l'application.
  • Nous n'exploitons aucun système de compte, et il n'y a aucune synchronisation cloud de vos données d'enseignement.
  • L'application effectue exactement un type d'appel réseau — pour valider votre code d'accès ou votre essai gratuit — décrit ci-dessous. Aucune donnée d'élève ou de groupe n'y est jamais incluse.
  • Hessati ne contient aucune publicité ni aucun SDK tiers d'analyse ou de suivi.

Données que vous saisissez dans l'application

Les informations suivantes sont stockées uniquement en local sur votre appareil, dans la base de données propre de l'application :

DonnéeUsageLieu de stockage
Noms des groupesOrganiser les élèves par classe/coursSur l'appareil uniquement
Noms des élèves et numéros de téléphone (optionnels)Identifier les élèves ; appeler un parent/élève en un tapSur l'appareil uniquement
Présences (dates de séance, présent/absent)Calculer le solde de chaque élèveSur l'appareil uniquement

Aucune de ces données n'est transmise où que ce soit, sauf si vous décidez volontairement de les exporter (voir « Sauvegarde et export » ci-dessous). Nous, le développeur, ne les recevons jamais et n'avons aucun moyen d'y accéder à distance.

Sauvegarde et export (votre action, votre choix)

Hessati vous permet d'exporter une sauvegarde complète (JSON) ou un tableur (CSV/Excel) de vos données via le sélecteur de fichiers standard d'Android. Vous choisissez la destination — stockage local, carte SD, un espace cloud, email, etc. Cela reste entièrement sous votre contrôle ; l'application elle-même ne téléverse ce fichier nulle part. Importer une sauvegarde remplace la base de données locale de l'application par le contenu du fichier, toujours entièrement sur l'appareil.

Validation du code d'accès / essai gratuit (le seul appel réseau)

Hessati nécessite un code d'accès fourni par le développeur (ou un essai gratuit unique de 30 jours) pour déverrouiller l'application, car elle est distribuée directement par le développeur plutôt que via un abonnement Play Store. Pour vérifier si un code ou un essai est valide, l'application contacte notre serveur de validation avec uniquement :

  • Le code d'accès que vous avez saisi (ou une demande de démarrage d'essai), et
  • Un identifiant d'appareil, utilisé uniquement pour appliquer la règle d'un code par appareil (ou un nombre limité d'appareils pour les codes multi-appareils) et pour empêcher qu'un même appareil ne réclame plus d'un essai gratuit.

Aucune donnée de groupe, d'élève, de présence ou de paiement n'est jamais incluse dans cet appel. Pendant un essai gratuit actif ou avec un code déjà validé, l'usage quotidien de l'application reste entièrement hors ligne, à l'exception de cette vérification périodique.

Autorisations demandées par l'application

  • Notifications — pour vous alerter localement lorsqu'un élève dépasse le seuil de solde que vous avez configuré. Il s'agit d'une notification locale, sur l'appareil ; rien n'est envoyé à un serveur pour la déclencher.
  • Stockage / accès aux fichiers (via le sélecteur de fichiers du système) — uniquement lorsque vous choisissez d'exporter ou d'importer une sauvegarde ou un tableur.
  • Téléphone / composeur — uniquement lorsque vous appuyez sur le numéro de téléphone d'un élève pour passer un appel ; l'application ne lit pas votre journal d'appels ni vos contacts.
  • Réseau — utilisé exclusivement pour l'appel de validation du code d'accès/essai décrit ci-dessus.

Partage des données

Nous ne vendons, ne louons, ni ne partageons vos données d'enseignement (groupes, élèves, présences, paiements) avec des tiers, car nous ne les recevons jamais en premier lieu. Les seules données qui atteignent notre serveur sont le code d'accès et l'identifiant d'appareil décrits ci-dessus, utilisés uniquement pour faire fonctionner le système de licence.

Conservation et suppression des données

Vos données d'enseignement résident dans la base de données locale de l'application jusqu'à ce que vous les supprimiez — en archivant ou supprimant des groupes/élèves dans l'application, ou en désinstallant l'application, ce qui efface entièrement sa base de données locale (sauf si vous avez exporté une sauvegarde, qui est votre propre copie séparée). Les enregistrements de code d'accès/appareil sur notre serveur de validation ne sont conservés que le temps nécessaire à la gestion de votre abonnement ou essai.

Confidentialité des mineurs

Hessati est un outil destiné aux enseignants/professeurs, pas une application destinée aux enfants. Un enseignant peut saisir le nom et le numéro de téléphone d'un élève mineur à des fins de suivi personnel ; ces données sont traitées comme toute autre donnée décrite ci-dessus (stockées localement, jamais transmises par nous).

Modifications de cette politique

سياسة الخصوصية — حصتي

آخر تحديث: 23 يوليو 2026

حصتي ("التطبيق") هو تطبيق لتتبع الحضور والمدفوعات يعمل دون اتصال بالإنترنت، مصمم للمدرسين الخصوصيين ومدرّسي المجموعات الصغيرة. توضح هذه السياسة البيانات التي يتعامل معها التطبيق، وربما الأهم من ذلك، ما لا يتعامل معه.

ملخص

  • يعمل حصتي دون اتصال بالإنترنت بالكامل. تبقى جميع بيانات المجموعات والطلاب والحضور والمدفوعات التي تُدخلها على جهازك، في قاعدة البيانات المحلية الخاصة بالتطبيق.
  • لا نُشغّل أي نظام حسابات، ولا توجد أي مزامنة سحابية لبيانات تدريسك.
  • يقوم التطبيق بنوع واحد فقط من الاتصال بالشبكة — للتحقق من رمز الوصول أو الفترة التجريبية — كما هو موضح أدناه. لا تُدرج أي بيانات طالب أو مجموعة في هذا الاتصال أبدًا.
  • لا يحتوي حصتي على أي إعلانات ولا أي أدوات تحليل أو تتبع من طرف ثالث.

يتيح لك حصتي تصدير نسخة احتياطية كاملة (JSON) أو جدول بيانات (CSV/Excel) لبياناتك باستخدام منتقي الملفات القياسي في أندرويد. أنت من يختار الوجهة — التخزين المحلي، بطاقة SD، مساحة تخزين سحابية، البريد الإلكتروني، إلخ. يبقى هذا الأمر بالكامل تحت سيطرتك؛ التطبيق نفسه لا يرفع هذا الملف إلى أي مكان. استيراد نسخة احتياطية يستبدل قاعدة بيانات التطبيق المحلية بمحتوى الملف، وهذا أيضًا يتم بالكامل على الجهاز.

التحقق من رمز الوصول / الفترة التجريبية (الاتصال الشبكي الوحيد)

يتطلب حصتي رمز وصول صادر عن المطوّر (أو فترة تجريبية مجانية لمرة واحدة مدتها 30 يومًا) لفتح التطبيق، نظرًا لأنه يُوزَّع مباشرة من قبل المطوّر بدلًا من اشتراك عبر متجر Play. للتحقق مما إذا كان الرمز أو الفترة التجريبية صالحة، يتصل التطبيق بخادم التحقق الخاص بنا مع تضمين ما يلي فقط:

  • رمز الوصول الذي أدخلته (أو طلب بدء فترة تجريبية)، و
  • معرّف الجهاز، يُستخدم فقط لتطبيق قاعدة رمز واحد لكل جهاز (أو عدد محدود من الأجهزة بالنسبة للرموز متعددة الأجهزة) ولمنع مطالبة الجهاز نفسه بأكثر من فترة تجريبية مجانية واحدة.

لا تُدرج أي بيانات عن المجموعات أو الطلاب أو الحضور أو المدفوعات في هذا الاتصال أبدًا. أثناء فترة تجريبية نشطة أو رمز تم التحقق منه مسبقًا، يبقى الاستخدام اليومي للتطبيق دون اتصال بالكامل باستثناء هذا التحقق الدوري.

الأذونات التي يطلبها التطبيق

  • الإشعارات — لتنبيهك محليًا عند تجاوز أحد الطلاب لحد الرصيد الذي حددته. هذا إشعار محلي على الجهاز؛ لا يُرسل شيء إلى خادم لتفعيله.
  • التخزين / الوصول إلى الملفات (عبر منتقي ملفات النظام) — فقط عند اختيارك تصدير أو استيراد نسخة احتياطية أو جدول بيانات.
  • الهاتف / الاتصال — فقط عند النقر على رقم هاتف طالب لإجراء مكالمة؛ لا يقرأ التطبيق سجل مكالماتك ولا جهات اتصالك.
  • الشبكة — تُستخدم حصريًا لاتصال التحقق من رمز الوصول/الفترة التجريبية الموضح أعلاه.

مشاركة البيانات

نحن لا نبيع ولا نؤجر ولا نشارك بيانات تدريسك (المجموعات، الطلاب، الحضور، المدفوعات) مع أي طرف ثالث، لأننا لا نتلقاها أصلًا. البيانات الوحيدة التي تصل إلى خادمنا هي رمز الوصول ومعرّف الجهاز الموضحان أعلاه، وتُستخدم فقط لتشغيل نظام الترخيص.

الاحتفاظ بالبيانات وحذفها

تبقى بيانات تدريسك في قاعدة البيانات المحلية للتطبيق إلى أن تحذفها — عن طريق أرشفة أو حذف المجموعات/الطلاب داخل التطبيق، أو عن طريق إلغاء تثبيت التطبيق، مما يمحو قاعدة بياناته المحلية بالكامل (ما لم تكن قد صدّرت نسخة احتياطية، وهي نسخة منفصلة خاصة بك). تُحفظ سجلات رمز الوصول/الجهاز على خادم التحقق لدينا فقط للمدة اللازمة لإدارة اشتراكك أو فترتك التجريبية.

خصوصية القُصَّر

حصتي أداة موجهة للمدرسين/المعلمين، وليس تطبيقًا موجهًا للأطفال. قد يُدخل المعلم اسم ورقم هاتف طالب قاصر لأغراض حفظ سجلاته الخاصة؛ تُعامَل هذه البيانات كأي بيانات أخرى موضحة أعلاه (تُخزَّن محليًا، ولا تُنقل من قبلنا أبدًا).
